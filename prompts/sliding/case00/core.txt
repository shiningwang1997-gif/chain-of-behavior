## **Step 1**

* **Input:** Current board `Board`, target number `1`, target position `(0,0)`, empty space position.
* **Output:** The minimal rectangle used for rotation computation.
* **Execution:**

  1. **Determine the rectangle:**

     * Construct the **smallest rectangle** that **must include all three points**:

       * Current position of the target number
       * Target position `(0,0)`
       * Current position of the empty space

     * Record the rectangle boundaries as `(r0, r1, c0, c1)`

     * The rectangle must satisfy:

       `r1 - r0 ≥ 1` and `c1 - c0 ≥ 1` (rectangle must be at least 2×2)

---

## **Step 2**

* **Input:** Rectangle coordinates `(r0, r1, c0, c1)` and the current board `grid`.
* **Output:** Ordered outer-layer coordinate list `P`.
* **Execution:**

  * Use the top-left of the board as the `(0,0)` coordinate system.

  * Enumerate the rectangle **outer layer in clockwise order**, starting from the top-left corner `(r0, c0)`:

    1. From `(r0, c0)` move right to `(r0, c1)`
    2. From `(r0, c1)` move down to `(r1, c1)`
    3. From `(r1, c1)` move left to `(r1, c0)`
    4. From `(r1, c0)` move up back toward `(r0, c0)` (excluding the starting corner)

  * Store the coordinates in order as `P`.

---

## **Step 3**

* **Input:** Outer-layer index list `P`, target number index `idx_d`, target position index `idx_t`.
* **Output:** Clockwise steps `cw_steps` and counter-clockwise steps `ccw_steps`.
* **Execution:**

  * Locate:

    * `idx_d`: the index of the target number in `P`
    * `idx_t`: the index of the target position `(0,0)` in `P`

  * Compute:

    * `cw_steps = (idx_t - idx_d) % len(P)`
    * `ccw_steps = (idx_d - idx_t) % len(P)`

---

## **Step 4**

* **Input:** `cw_steps`, `ccw_steps`, relative row/column relationship between the target number and the target position, and the empty space position.
* **Output:** Locked rotation direction `direction_locked` and locked rotation steps `steps_locked`.
* **Execution:**

  * If `cw_steps < ccw_steps`:

    * `direction_locked = "CW"`
    * `steps_locked = cw_steps`

  * If `ccw_steps < cw_steps`:

    * `direction_locked = "CCW"`
    * `steps_locked = ccw_steps`

  * If `cw_steps == ccw_steps`, apply tie-breaking rules:

    * **Same row:**

      * Target number to the **right** of target position → `"CW"`
      * Target number to the **left** of target position → `"CCW"`

    * **Same column:**

      * Target number **above** the target position → `"CW"`
      * Target number **below** the target position → `"CCW"`

    * **Diagonal case:**

      * Assume one step of `"CW"` and one step of `"CCW"`
      * Choose the direction that **minimizes the Manhattan distance** between the empty space and the target number

    * In tie cases:

      * `steps_locked = cw_steps`

---

## **Step 5 — Final Output**

* **Input:** `direction_locked`, `steps_locked`, rectangle `(r0, r1, c0, c1)`
* **Output:** A JSON object containing **rotation direction, step count, and rectangle boundaries**.
* **Execution:**

> After completing all reasoning and step-by-step calculations above, **always output exactly one JSON object** with the final rotation direction, step count, and rectangle boundaries.  
> You may include reasoning steps, explanations, and intermediate calculations above, but the **last output must be the JSON object**.

**Format:**

```json
{
  "direction": "CW",   // or "CCW"
  "steps": steps_locked,
  "rectangle": [r0, r1, c0, c1]
}
```
